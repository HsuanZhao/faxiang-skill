# 法官造像 · 中国法官思维蒸馏器

> 法官不发微博，不开播客，不接受"人生访谈"。他/她唯一持续、公开、可信的自我表达，是判决书里的说理段落，和期刊上的署名文章。

## 核心理念

法官造像不是复制判决结论，是**提炼裁判方法论**：

- 他/她用什么**法律解释方法**处理疑难问题？（文义/体系/目的/合宪性解释的偏好）
- 他/她面对法律漏洞、价值冲突时的**利益衡量路径**是什么？
- 他/她在裁判文书中如何**说理**？（论证结构、援引偏好、语言风格）
- 他/她**绝不会**采纳的论证方式是什么？（方法论反模式）
- 什么是这个Skill**做不到的**？（诚实边界——尤其是"公开论述 ≠ 内心真实心证"的鸿沟）

**关键区分**：捕捉的是这位法官**怎么说理**，不是他判过什么案、判决结果是什么。蒸馏对象是方法论，不是"预测某类案件会怎么判"。

---

## 前置红线（先读，比流程更重要）

在开始任何调研之前，必须向用户明确以下几点，这是本skill区别于通用人物蒸馏的核心约束：

1. **不做"预测判决结果"的Skill**。任何暗示"这位法官会怎么判某个具体案子"的产出方向都要拒绝或重新框定为"基于其一贯说理方法进行的学术推演，不代表任何具体案件的裁判意见"。
2. **不冒充法官身份发表评论**。生成的Skill在角色扮演时必须清楚标注"以下是基于公开学术文献对XX法官法律方法论的模拟推演，不代表XX法官本人观点，更不构成任何案件的裁判意见或法律意见"。
3. **在任法官与退休法官的处理方式不同**：
   - 在任法官：只使用其以个人名义公开发表的学术成果，不涉及其承办案件的裁判文书内容分析（除非该文书本身就是其发表的案例评析素材，且已在裁判文书网公开、不涉及未生效或敏感案件）
   - 退休/离任法官：可适度扩展至其公开的回忆录、访谈、荣休致辞等
4. **法院整体立场 ≠ 法官个人观点**：论文中出现的"某某法院经审判委员会讨论认为"等表述属于法院集体意见，不能等同于该法官个人思维方式，调研时必须标注区分。

## 执行流程

### Phase 0A: 需求澄清

收到法官姓名后确认：

1. **身份核实**：姓名 + 现任/曾任法院及庭室（如"XX法官，现任/曾任最高人民法院民二庭审判长"）。中国法官同名概率较高，务必核实清楚，避免张冠李戴
2. **在任状态**：在任 / 已退休或离任 / 已故。决定信息源边界（见前置红线第3条）
3. **聚焦方向**（可选）：全面法律方法论画像 vs 聚焦某一领域（如民商事疑难案件裁判方法、数字法治/网络法专题、执行程序思路等）
4. **用途**：法律写作参考？案例研讨会角色扮演？法律方法论教学？——用途会影响诚实边界的写法，务必问清楚，尤其要排除"用于预测具体案件结果"的用途
5. **新建 or 更新**：检查 `.claude/skills/` 目录是否已有该法官的Skill
6. **本地语料**：「您手上有没有这位法官的一手素材？比如其专著、论文合集PDF、审判委员会/法官会议纪要（如合规获得）、学术会议论文集、其指导案例的公开评析文章？有的话直接提供，比公开检索的质量高得多，也更能规避检索盲区。」
7. **蒸馏档位**：同通用版本，告知成本量级：

| 档位 | 调研规模 | 适用场景 |
|------|---------|---------|
| 快速 | 3个维度（学术论文+裁判说理+案例评析），每维度限5篇 | 先试试效果 / 该法官公开论著较少 |
| 标准（默认） | 6个维度完整调研 | 大多数场景 |
| 深度 | 6维度 + 尽量下载全部可获得的一手论著全文存档 | 学术研究 / 法律方法论教学用途 |

用户只给了姓名没有更多信息 → 默认标准档 + 全面方法论画像，但**必须先反问身份核实与在任状态**，这两项不给默认值，因为关系到信息源合规边界。

确认后 → 跳到 Phase 0.5。

#### 每个Agent的硬性要求

- 调研结果必须写入 `references/research/0X-xxx.md`
- **每条信息必须标注**：期刊全称+年份+期号（如《法律适用》2021年第6期）、是否为核心/CLSCI来源、一手（本人署名文章）or 二手（他人转述评价）
- 严格区分「法官本人的学术观点」vs「法院集体意见（如审委会讨论意见）」vs「他人对其观点的转述」
- 发现同一法官在不同时期论述有变化或矛盾之处，如实记录，不要强行统一
- **涉及具体案件时**，只记录该案件在法官本人发表的学术文章中被作为"案例素材"讨论的部分，不额外调研该案件的舆论评价或社会影响

#### 信息源优先级（法官专属）

| 来源类型 | 揭示什么 | 权重 |
|---------|---------|------|
| **用户提供的一手素材**（专著全文、内部培训讲稿等） | 完整原文，未经二手过滤 | **最高+** |
| 《法律适用》《人民司法》《数字法治》署名文章 | 法官对具体司法实务问题的系统性思考，这三本刊物均为法院系统权威业务期刊，法官投稿意愿与真实度较高 | 最高 |
| 全国法院学术讨论会获奖论文 | 经同行评审认可的法律方法论水平，代表性强 | 最高 |
| 个人专著 | 系统性法律思想 | 最高 |
| 其他CLSCI核心法学期刊论文 | 学术共同体内的严谨论证 | 高 |
| 本人撰写的指导性案例评析、案例解读 | 法律适用方法在具体案件中的展开 | 高 |
| 裁判文书说理部分（仅限公开合规文书） | 实际裁判中的论证风格，但需注意可能经合议庭讨论润色，未必完全体现个人风格 | 中高 |
| 学术会议发言纪要（如有公开记录） | 即席学术观点 | 中等 |
| 同行学术引证与书评 | 外部视角、学术定位 | 中等 |
| 二手媒体报道、内部工作总结转述 | 参考但需谨慎核实 | 低 |

#### 信息源黑名单（永远排除）

- **知乎、微信公众号**：洗稿严重、信息失真率高，且法律类自媒体常有断章取义裁判文书的问题
- **百度百科/百度知道**：信息陈旧且不可靠
- **来源不明的判决书截图、网络流传的"内部纪要"**：无法核实真伪，且可能涉及未生效裁判或违反审判秘密
- **涉及具体在办案件的社会舆论报道**：不属于法官本人学术观点，且可能影响司法独立评价的中立性

#### 权威中文渠道白名单（法官专属）

- 学术期刊：《法律适用》《人民司法》《数字法治》，及知网/万方检索到的其他CLSCI核心法学期刊
- 会议与评奖：全国法院学术讨论会（国家法官学院主办）历年获奖论文及名单、各高级人民法院学术研讨会
- 裁判文书：中国裁判文书网（wenshu.court.gov.cn）、北大法宝、威科先行——仅用于已生效、可公开检索的文书
- 官方履历信息：最高人民法院、各级法院官网的法官简介栏目（如公开）
- 出版社官方书目：人民法院出版社、法律出版社、中国法制出版社等

#### 工具辅助：裁判文书与学术检索脚本

以下脚本存放于 `.claude/skills/faguan-fangfalun/scripts/`，各Agent按需调用。**总原则**：裁判文书网、知网等平台都有反爬机制和明确的服务条款限制，脚本不做自动化抓取、不破解验证码、不绕过付费墙，只负责"辅助定位+规范化人工核实后的结果"。

##### 1. 裁判文书——本地手动下载语料模式（默认推荐路径）

裁判文书网检索受限较多，**更推荐用户在裁判文书网/北大法宝/威科先行等平台手动搜索并下载文书原文，本地打包后交给Agent读取分析**，而不是让Agent自己联网检索。这样也天然规避了自动化抓取的合规风险。

**目录结构**（放在 `references/sources/judgments/` 下）：

```
references/sources/judgments/
├── manifest.json          # 核心元数据清单，Agent 2只读取这里标记为已通过校验的条目
├── raw/                   # 用户手动下载的原始文件，落地到这里（未校验状态）
│   ├── (2021)京01民终1234号.pdf
│   ├── (2020)沪02民终567号.docx
│   └── ...
├── verified/              # 校验通过后自动移入，Agent实际读取这个目录下的文件
│   └── (2021)京01民终1234号.pdf
└── excluded/              # 未通过校验的文件连同排除原因一起归档，便于追溯
    ├── (2019)粤03民终89号.pdf
    └── excluded-log.json
```

**manifest.json 记录格式**（每份文书一条）：

```json
{
  "filename": "(2021)京01民终1234号.pdf",
  "doc_number": "(2021)京01民终1234号",
  "court": "北京市第一中级人民法院",
  "judge_name": "张三",
  "role_in_case": "审判长",           // 审判长/承办法官/合议庭成员-非主审
  "case_type": "民事二审",
  "year": 2021,
  "is_final_effective": true,        // 必须为true才可采纳
  "verified_by_human": true,         // 必须为true才可采纳
  "sensitive_content_check": "已核对，无未成年人/隐私/涉密内容",
  "notes": ""
}
```

**采纳硬性条件**：`is_final_effective=true` 且 `verified_by_human=true` 且已核对无敏感内容，三者同时满足才进入 `verified/`，否则一律进 `excluded/` 并记录排除原因。**非主审法官**（合议庭成员但非承办人/审判长）的文书即使采纳，权重也要标注为"仅作参考"，不能与主审文书同等对待。

**脚本**：`scripts/wenshu_lookup_helper.py` 提供三个子命令，把上述流程自动化：

```bash
# 1. 用户先把手动下载的文书原文放进 references/sources/judgments/raw/

# 2. 扫描raw/下的新文件，逐个交互式询问元数据（法院/角色/是否生效/是否核实等）
python3 wenshu_lookup_helper.py scan [skill目录]

# 3. 根据录入结果，自动把文件分流到 verified/ 或 excluded/
python3 wenshu_lookup_helper.py sort [skill目录]

# 4. 查看当前可供Agent 2读取分析的文书清单（附主审/非主审权重提示）
python3 wenshu_lookup_helper.py list-verified [skill目录]
```

- 每个Agent检索时，先用**窄检索式**（姓名+法院全称精确匹配）排除同名干扰，再逐步放宽

#### 失败模式与降级路径

| 触发条件 | 一线修复 | 仍失败兜底 |
|---------|---------|-----------|
| 该法官公开论著数量很少（<5篇） | Phase 0.5即告知用户，心智模型/方法论要点减至2-3个 | 诚实边界大篇幅标注"该法官学术产出有限，本Skill仅能反映其部分公开观点" |
| 无法核实身份，存在同名法官 | 立即暂停，向用户确认具体法院/庭室/任职时间 | 用户无法提供更多信息 → 不予蒸馏，说明原因 |
| 该法官在任且敏感案件较多 | 严格排除具体案件裁判结果的讨论，只留学术论文和抽象方法论 | 如仍不确定边界，向用户说明并建议缩小范围至纯学术论文 |
| 裁判文书来源无法核实公开合规性 | 该条信息不采纳 | 完全依赖学术论文与公开案例评析，跳过裁判文书说理维度 |

**关键规则**：宁可交付一个诚实标注"该法官公开学术信息有限"的60分Skill，也不要用来源不明的裁判文书内容拼凑出一个看似丰富实则违规的Skill。

### Phase 2: 框架提炼（法律方法论版）

#### 2.1 法律方法论要素提取（3-7个）

将通用版"心智模型"替换为**法律方法论要素**，每个要素需满足：
- **跨案复现**：在≥2篇不同主题的论文/案例评析中都体现同一方法论倾向？
- **可推断性**：能推断此法官面对同类型新的疑难法律问题时的分析路径？
- **区分度**：不是所有法官处理该类问题都会这样论证？

三重通过 → 列为方法论要素；仅1-2重 → 降级为"论证习惯"；0重 → 丢弃。

每个要素记录：名称、一句话描述、来源证据（≥2篇文献/文书）、典型应用场景、局限（该方法论在什么情况下不适用或该法官自己也曾偏离）。

#### 2.2 裁判说理习惯提取（5-10条）

= 此法官组织论证时的固定模式。如"先穷尽文义解释可能性，再诉诸体系解释""偏好援引指导性案例而非单纯学说"等，需有具体文献/文书佐证。

#### 2.3 学术表达风格分析

| 维度 | 提取内容 |
|------|---------|
| 论证结构偏好 | 争议焦点前置 or 后置、演绎为主 or 类型化归纳为主 |
| 术语与概念 | 高频使用的法律术语、自创或重点阐释的概念 |
| 援引偏好 | 偏好援引法条/司法解释/指导性案例/域外学说/本土学说的比例 |
| 语言风格 | 严谨克制型 or 说理充分展开型、是否常用比喻或类比 |
| 不确定性表达 | 对疑难问题是"倾向性意见"型还是"结论明确"型 |

#### 2.4 价值取向与方法论边界

- **价值取向**：3-5条其在裁判/论述中体现的价值排序（如法的安定性 vs 个案正义、形式理性 vs 实质理性的权衡倾向）
- **方法论反模式**：此法官明确批评或从不采纳的论证方式
- **内在张力**：如"强调严格法条主义"与"某些论文中又主张适度能动司法"之间的张力——如实记录，不要抹平

#### 2.5 学术谱系

此法官的学说渊源受谁影响（哪位学者/哪个学派）→ 在法学方法论光谱上的定位（如偏概念法学 or 利益法学 or 评价法学倾向）。

#### 2.6 诚实边界（法官版，必须包含以下条目）

- **本Skill不代表[法官姓名]本人观点，更不构成任何案件的裁判意见或法律意见**
- 不能预测该法官对具体案件的裁判结果，只能模拟其一贯的法律方法论
- 公开学术论述 ≠ 内心真实心证，法官在判决书中的说理也受合议庭讨论、审级监督等因素影响，未必完全是个人观点的直接呈现
- 信息截止到调研时间点，且仅覆盖其公开发表的学术成果
- 若该法官公开论著数量有限，需明确标注方法论要素的推测成分

## 品味守则（速查）

| 原则 | 一句话 |
|------|--------|
| 学术论文 > 只言片语 | 一篇3000字以上的期刊论文比任何转述都更能体现方法论 |
| 方法论 > 判决结果 | 蒸馏的是"怎么推理"，不是"判了什么" |
| 承认局限 > 假装全面 | 该法官公开论著少，就诚实标注，不要靠推测填满 |

### ❌ 反模式黑名单

| # | 反模式 | 为什么 / 替代做法 |
|---|--------|------------------|
| 1 | 用Skill预测具体在办案件的裁判结果 | 违反司法独立原则，也超出学术推演的合理边界，必须拒绝 |
| 2 | 把法院集体意见当作法官个人观点 | 审委会讨论意见、合议庭意见不等于个人思维方式，必须标注区分 |

| 6 | 忽略该法官论述中的矛盾或立场变化 | 内在张力是真实性的来源，抹平反而失真 |
| 7 | 生成的Skill缺少"不代表本人观点"的免责声明 | 涉及法官个人名誉与司法公信力，免责声明是硬性要求，不可省略 |

## 特殊场景

### 在任 vs 退休/离任法官

- **在任法官**：严格限定在学术论文、专著、公开案例评析范围内，不涉及具体承办案件的舆论评价，方法论要素表述需格外克制
- **退休/离任法官**：可适度扩展至回忆录、荣休致辞、公开访谈（如接受权威媒体如《人民法院报》等采访），但仍需遵循"不预测具体案件"的原则

### 学术产出极少的法官（可用来源<10篇）

1. Phase 0.5即告知用户「该法官公开学术信息有限，生成的Skill质量会受限」
2. 方法论要素减至2-3个，均标注"基于有限公开信息推测"
3. 诚实边界大幅加强，明确列出信息不足的具体维度
4. 建议用户询问该法官本人或其所在法院是否愿意提供论文合集等一手素材（需征得本人同意）

## 最后

法官造像蒸馏的不是判决结果，是说理的骨架。

一个好的法官方法论Skill，帮助法律从业者和研究者理解"这类疑难问题可以怎样被严谨地论证"，而不是预测"某个案子会怎么判"。这是本Skill与通用人物蒸馏最根本的区别，也是使用它时必须始终守住的边界。
